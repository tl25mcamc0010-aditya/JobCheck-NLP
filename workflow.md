# Project Workflow

This document describes the end-to-end workflow for the **Fake Job Detector** project. It covers data ingestion, preprocessing, feature engineering, model training, inference, and the web-based UI with batch + analytics.

---

## 1) Data (Input)

- **Dataset (training)**: `Data/fake_job_postings.csv`
  - Contains job postings with fields such as `description`, `fraudulent`, and other job metadata.

- **Prediction history (runtime)**: `Database/jobs.db`
  - SQLite database used by the web UI to store each prediction.
  - Schema (managed by `src/database.py`):
    - `id` (int, primary key)
    - `description` (text)
    - `prediction` (text)
    - `created_at` (datetime)

---

## 2) Preprocessing (Text Cleaning)

**File**: `src/preprocessing.py`

- Uses **NLTK** to:
  - Download required data (`stopwords`, `wordnet`).
  - Remove stopwords.
  - Lemmatize tokens.
- Performs basic cleaning:
  - Lowercasing
  - URL removal
  - Removing non-alphabet characters

**Key function**:

- `clean_text(text)` → returns cleaned text string.

> Note: `preprocessing.py` currently reads the dataset on import, but training and prediction only use `clean_text()`.

---

## 3) Feature Engineering (Vectorization)

**File**: `src/feature_engineering.py`

- Uses `TfidfVectorizer` to turn cleaned job descriptions into numeric feature vectors.
- Configuration:
  - `max_features=5000`
  - `ngram_range=(1, 2)`

**Key function**:

- `create_tfidf(text)` → returns `(X, vectorizer)` where `X` is the TF-IDF matrix.

---

## 4) Model Training

**File**: `src/train_model.py`

Workflow:

1. Load `Data/fake_job_postings.csv` into a DataFrame.
2. Clean text using `clean_text()` and store in `df["clean_text"]`.
3. Create TF-IDF features using `create_tfidf()`.
4. Split into train/test sets using `train_test_split()`.
5. Train a `LogisticRegression` model.
6. Print test accuracy.
7. Save out:
   - Model → `Models/model.pkl`
   - TF-IDF vectorizer → `Vectorizers/tfidf.pkl`

---

## 5) Prediction (Inference)

### Library API

**File**: `src/predict.py`

Workflow:

1. Load saved model (`Models/model.pkl`) and vectorizer (`Vectorizers/tfidf.pkl`).
2. Clean input text via `clean_text()`.
3. Transform the cleaned text with the vectorizer.
4. Predict with the model.
5. Return human-readable label:
   - `1` → "Fake Job"
   - `0` → "Real Job"

**Key function**:

- `predict_job(text)` → returns `"Fake Job"` / `"Real Job"`.

---

## 6) Web UI (Flask)

**File**: `app.py`

The Flask application provides:

- **Homepage (`/`)**
  - Displays the prediction form, recent predictions, and dashboard stats.
- **Predict (`/predict`)**
  - Classifies submitted job descriptions.
  - Stores the prediction in the SQLite DB (`Database/jobs.db`).
- **History (`/history`)**
  - Shows all stored prediction records.
- **Stats (`/stats`)**
  - Displays aggregate metrics + charts (real vs fake, trend over time).
- **Batch Upload (`/batch`)**
  - Accepts a CSV file containing a `description` column.
  - Runs predictions for each row and stores results in the DB.
  - Provides a download endpoint (`/batch/download`) to export batch results as CSV.

Templates:

- `templates/index.html` — main detector UI
- `templates/history.html` — prediction history view
- `templates/stats.html` — analytics dashboard
- `templates/batch.html` — batch upload view

---

## 7) Running the Project

### Setup

1. Install dependencies:

   ```bash
   pip install -r requirements.txt
   ```

   _(If `requirements.txt` does not exist, install: `pandas scikit-learn nltk flask`.)_

2. (Optional) Download NLTK data:
   ```python
   import nltk
   nltk.download('stopwords')
   nltk.download('wordnet')
   ```

### Train the model

```bash
python -m src.train_model
```

### Run the web UI

```bash
python app.py
```

Visit `http://127.0.0.1:5000/` to use the UI.

---

## 8) Outputs

- `Models/model.pkl` — trained classification model
- `Vectorizers/tfidf.pkl` — saved TF-IDF vectorizer
- `Database/jobs.db` — persisted prediction history
- (Optional) `Data/predictions.csv` — not generated by the current web UI; can be produced by exporting batch results
